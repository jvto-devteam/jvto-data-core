## Criteria deliberately NOT met — and why (no fake-green)

- **`by_channel` > 2 categories** and **`by_payment_status` > 1 value** are **not
  met.** `getBookingAnalytics()` derives both from record fields (`channel`,
  `payment_status`) at runtime. Populating them requires real source data —
  the channel id→name lookup (gap-2 Track A) and a payments source (gap-3) — both
  of which need the live `new-backoffice` MySQL that is unreachable from this
  environment. Fabricating those values would corrupt the ROI and cash-flow
  analysis these gaps exist to enable, so they were left authentic and documented
  as BLOCKED-pending-DB (see gap-2 / gap-3 diagnoses).
